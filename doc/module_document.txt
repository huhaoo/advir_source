### class haze_degradation

module_name: haze_degradation
location: /home/huhao/adv_ir/source/degradation.py
class_name: haze_degradation (nn.Module)

hyperparameters:
- config_dataclass: HazeDegradationConfig
- map_config: ControlMapConfig
- airlight_init
- beta_mean, beta_std (optional), min_beta

parameter:
- internal module: control_map (for spatial beta base)
- fixed buffer: airlight with shape (1, 1, 1, 1) or (1, C, 1, 1), initialized by airlight_init

forward_method:
- signature: forward(image, distance_map) -> Tensor
- image_shape: (B, C, H, W)
- distance_map_shape: (B, 1, H, W) or (B, C, H, W)
- formula: t = exp(-beta * d), y = x * t + A * (1 - t)
- beta map behavior:
	- when beta_std is provided: normalize base map then apply std/mean control
	- when beta_std is None: disable variance control and use beta = (base_beta - mean(base_beta)) + beta_mean
- output_shape: (B, C, H, W)

losses:
- get_regularization_loss: from internal control_map
- get_regularization_items: from internal control_map

### function random_degradation_configs_from_image

module_name: random_degradation_configs_from_image
location: /home/huhao/adv_ir/source/degradation.py
class_name: function

purpose:
- given one image tensor, sample a valid triplet of Noise/Rain/Haze dataclass configs

input:
- image shape: (B, C, H, W) or (C, H, W)

sampling_rules:
- noise_strength in {15, 25, 50} / 255
- rain router num_maps in {4, 8, 16}
- haze airlight base in [0.85, 1.0], per-channel jitter +-0.02, channel clamp to [0.85, 1.0]
- haze beta_mean in [0.1, 0.5]
- haze beta_std is sampled as beta_mean * (0.05 + 0.25 * U[0,1])

return:
- (NoiseDegradationConfig, RainDegradationConfig, HazeDegradationConfig)

### class promptir_adv_mix_dataset

module_name: promptir_adv_training
location: /home/huhao/adv_ir/source/promptir_adv_training.py
class_name: promptir_adv_mix_dataset (torch.utils.data.Dataset)

purpose:
- wrap PromptIR train dataset and add adversarial samples by ratio
- support periodic adversarial resampling every N epochs
- guarantee haze adversarial generation only when matched depth mat exists

inputs:
- base_dataset: PromptTrainDataset
- args fields:
	- adv_ratio
	- adv_resample_epochs
	- adv_steps1, adv_steps2
	- adv_step_size, adv_lambda_reg
	- adv_rain_topk
	- adv_max_side
	- adv_promptir_ckpt
	- adv_cache_root
	- adv_attack_device
	- adv_samples_per_resample

key_methods:
- resample_adversarial(epoch, force=False)
	- regenerate adversarial pool with tqdm progress
	- target size auto from ratio or fixed by adv_samples_per_resample
	- for each call, creates one lightweight folder: resample_epoch_xxxx/input and resample_epoch_xxxx/target
	- training reads only these paired image folders
- resample_main_process_then_sync(epoch, force=False)
	- in DDP, each rank generates an equal shard of adversarial samples
	- each rank writes one shard manifest under shards/
	- rank 0 merges shard manifests into one global manifest.json
	- all ranks synchronize and then load the same merged sample list
- __len__()
	- returns base_len + adv_len_ready
- __getitem__(idx)
	- base branch: returns original PromptTrainDataset sample
	- adv branch: returns cached adversarial pair with train-time crop/augmentation

policy:
- adversarial subset avoids noise-only mode
- if haze is enabled, corresponding RESIDE depth mat is mandatory

outputs:
- /home/huhao/adv_ir/PromptIR/data/Train/adv_pairs/shared/
	- resample_epoch_xxxx/input/*.png
	- resample_epoch_xxxx/target/*.png
	- resample_epoch_xxxx/shards/manifest_rank_xxxx.json
	- resample_epoch_xxxx/manifest.json

key_functions:
- load_promptir_model(checkpoint_path, device)
	- loads PromptIR checkpoint from Lightning-format state_dict
	- strips net.* prefix and loads into PromptIR(decoder=True)
	- sets eval mode and requires_grad=False for all PromptIR params
- build_default_controller_from_image(image, enable_count_probs=(0.5, 0.4, 0.1))
	- samples 1/2/3 active degradations with probability 50%/40%/10%
	- samples enabled subset before controller construction
	- samples rain/haze order once before construction
	- resulting controller uses fixed subset and fixed order during the whole inner-loop
- run_single_image_adversarial_degradation_search(...)
	- objective: maximize task_loss - lambda_reg * reg_loss
	- implementation: hierarchical GD
	- outer loop calls PromptIR steps1 times to get fixed d(task_loss)/d(x_deg)
	- inner loop updates degradation steps2 times with fixed outer gradient surrogate
	- expensive PromptIR call count is controlled by steps1
	- task_loss: L1(x_hat, target)
	- reg_loss: degradation_controller.get_regularization_loss()
	- only degradation controller params are optimized
- validate_random_selection_rules(...)
	- checks empirical 1/2/3 selection ratio and ordering constraints

run:
- entry: run_promptir_single_image_attack()
- run_command: conda run -n advir --no-capture-output python /home/huhao/adv_ir/source/promptir_attack.py --num_steps 6 --step_size 0.03 --lambda_reg 0.05 --device cpu --num_gpus 1 --max_side 256
- run-command (hierarchical): conda run -n advir --no-capture-output python /home/huhao/adv_ir/source/promptir_attack.py --steps1 4 --steps2 4 --step_size 0.03 --lambda_reg 0.05 --device cpu --num_gpus 1 --max_side 256
- behavior:
	- resets /home/huhao/adv_ir/tmp_demo before run
	- loads one RESIDE clear-depth pair
	- fixed policy uses rain+haze only (noise disabled)
	- uses max hyperparameters (rain num_maps=16, rain_topk=16, haze beta_mean max, airlight max)
	- runs inner-loop attack and saves original/initial/final degraded and restored images
	- saves step-wise logs, active degradation state, and random-rule statistics
	- supports multi-gpu forward with DataParallel when --device cuda and --num_gpus > 1

queue_mode:
- entry: run_promptir_multi_image_attack_queue()
- trigger: --queue_mode or --num_samples > 1
- behavior:
	- creates persistent worker processes (one process per GPU)
	- each worker loads PromptIR checkpoint once and then processes assigned samples sequentially
	- avoids repeated process startup / CUDA init / checkpoint loading between samples
	- output root: /home/huhao/adv_ir/tmp_demo/promptir_attack with per-sample subfolders and queue_summary.json
	- new flag: --queue_save_samples controls whether to save per-sample visual assets in queue mode
	- default queue behavior keeps --queue_save_samples off to reduce CPU/I/O stalls and improve GPU utilization
	- queue_summary.json includes timing fields: avg_total_sample_sec / avg_load_prepare_sec / avg_attack_and_optional_save_sec

regularization_strength_visual_run:
- purpose: compare lambda_reg effects with image-level visual outputs under fixed attack settings
- recommended command template:
	- conda run -n advir --no-capture-output python /home/huhao/adv_ir/source/promptir_attack.py --device cuda --num_gpus 2 --num_samples 2 --queue_mode --queue_save_samples --steps1 4 --steps2 4 --step_size 0.03 --lambda_reg <value> --max_side 256 --seed 123 --ckpt /home/huhao/adv_ir/PromptIR/train_ckpt_8192/epoch=63-step=114688.ckpt
- recommended lambda set: {0.00, 0.02, 0.05, 0.10, 0.20}
- per-sample key files for comparison:
	- worst_degraded.png (gradient ascent worst-case degradation)
	- worst_restored.png (PromptIR output under worst-case degradation)
	- attack_log.json (numeric trajectory and final metrics)
- persistent archived visual path (outside tmp_demo reset): /home/huhao/adv_ir/source/tmp_lambda_visual_demo
- current recommendation:
	- under default random settings, use lambda_reg=0.05 as the current balanced choice

outputs:
- /home/huhao/adv_ir/tmp_demo/promptir_attack
