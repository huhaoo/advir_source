path_mapping_rules:
- noisy:
  - each manifest item is already a clean image relative path (e.g., noise/mix/00001.bmp).
- rainy:
  - input path under `.../input/...` maps to clear path by replacing `/input/` with `/target/`.
- hazy:
  - hazy path under `.../haze/reside_ots/haze/xxxx_a_b.jpg` maps to clear path `.../clear/xxxx.jpg`.
  - depth path maps from clear path by `.../clear/xxxx.jpg -> .../depth/xxxx.mat`.

meaning_for_training:
- `promptir_clear_depth_sets_only.json` is the minimal loader-friendly file:
  - `splits.train.clear_paths`, `splits.val.clear_paths`, `splits.test.clear_paths`
  - `splits.*.depth_paths` (non-empty only for haze-related clear images)
- `promptir_clear_depth_splits.json` is the full diagnostic file:
  - includes per-task subgroups (`noisy`, `rainy`, `hazy`)
  - includes missing-path lists and detailed counts
- `promptir_clear_depth_splits_stats.json` is a compact summary:
  - counts and first-5 examples for quick sanity checks
